# KY-039 Heart Rate Sensor (Arduino Nano)

![Board](https://img.shields.io/badge/Board-Arduino%20Nano-00979D?logo=arduino&logoColor=white)
![IDE](https://img.shields.io/badge/IDE-Arduino%20IDE-00979D?logo=arduino&logoColor=white)
![License](https://img.shields.io/badge/License-MIT-yellow.svg)

A ready-to-use KY-039 heart rate sensor example for Arduino IDE, featuring practical signal processing designed for environmental light interference and inverted waveform scenarios:

- Raw signal acquisition
- EMA low-pass filtering
- DC offset tracking
- AC signal inversion and centering
- Threshold-triggered pulse wave output

This project is suitable for:

- Beginners working with KY-039 and Arduino for the first time
- Those who want to observe heart rate waveforms using Serial Plotter
- Those who want to quickly adjust threshold values (THRESHOLD) for real-time detection

## Features

- 4 visualizable data channels for synchronized observation in Serial Plotter
- Separate high-frequency noise and low-frequency background light using `ALPHA_SMOOTH` and `ALPHA_DC`
- Inversion strategy for typical heart rate signals that appear as troughs
- Fixed sampling interval with `delay(20)`, approximately 50Hz sampling rate

## Hardware Requirements

- Arduino Nano (ATmega328P)
- KY-039 Heart Rate Sensor Module
- Dupont wires
- USB cable

## Quick Start

1. Open Arduino IDE.
2. Load `KY039-HR to Arduino Nano code.ino`.
3. In `Tools > Board` select Arduino Nano.
4. In `Tools > Processor` select your Nano variant (commonly ATmega328P or Old Bootloader).
5. Select the correct port and upload.
6. Open `Tools > Serial Plotter`, set baud rate to `9600`.

## Output Data Format

Each line outputs 4 values separated by spaces:

1. `rawValue`: Raw sensor value
2. `smoothedValue`: EMA-filtered signal
3. `acSignal`: AC signal after DC removal and inversion (centered at 512)
4. `beatDetected`: Threshold-triggered pulse wave (0 or 512)

Example (illustrative):

```text
522 520 511 0
524 521 514 512
523 521 513 0
```

## Parameter Tuning

Priority adjustable constants:

- `ALPHA_SMOOTH` (default `0.2`)
  - Increase: More sensitive, but may be more noisy
  - Decrease: Smoother, but more latency
- `ALPHA_DC` (default `0.02`)
  - Increase: Tracks environmental light changes faster
  - Decrease: More stable baseline
- `THRESHOLD` (default `513`)
  - Check the peak height of `acSignal` to fine-tune
  - Too low: Easy to misdetect
  - Too high: Easy to miss detections

## Troubleshooting

- Cannot see clear waveform
  - Ensure finger is in firm contact with sensor, but not pressing too hard
  - Reduce direct environmental bright light exposure
  - Check if GND is properly connected and wiring is secure
- Waveform has too much noise
  - Lower `ALPHA_SMOOTH`
  - Slightly increase `THRESHOLD`
- Almost always triggering or not triggering at all
  - Observe the actual upper and lower bounds of `acSignal`, then adjust `THRESHOLD`

## Notes

- This project is for educational, experimental, and signal observation purposes only, not for medical use.
- Different fingers, pressure, skin conditions, and ambient light will all affect signal quality.
